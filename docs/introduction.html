Introduction

What is BOOSE?

BOOSE (Basic Object Orientated Software Engineering) is an educational programming language designed to teach fundamental programming concepts through visual output. BOOSE combines traditional programming constructs like variables, loops, and methods with a simple graphics system that provides immediate visual feedback.

Purpose and Design Philosophy

BOOSE was created as a learning tool to help students understand core programming concepts in an engaging, visual way. By drawing shapes and graphics, learners can immediately see the results of their code, making abstract programming concepts more concrete and accessible.

The language emphasizes:

  • Simplicity: Easy-to-understand syntax with minimal boilerplate
  • Visual Feedback: Every program produces visual output on a canvas
  • Progressive Learning: Start with basic drawing, progress to variables, control flow, and methods
  • Object-Oriented Principles: Commands follow object-oriented design patterns

Key Features

Drawing Commands

BOOSE provides built-in drawing capabilities:

  • Move the pen to any position on the canvas
  • Draw lines, circles, and rectangles
  • Set pen colors using RGB values
  • Write text to the canvas

Programming Constructs

BOOSE supports standard programming features:

  • Variables: Integer, real, boolean, and array types
  • Expressions: Mathematical operations and comparisons
  • Control Flow: If/else statements, while loops, and for loops
  • Methods: Define reusable functions with parameters and return values
  • Arrays: Fixed-size arrays with poke/peek operations

Example Program

Here's a simple BOOSE program that draws colored shapes:

int radius = 50
int width = 100

pen 0,0,255
moveto 100, 100
circle radius

pen 255,0,0
moveto 150, 50
rect width, 100

This program:

  1. Declares integer variables for radius and width
  2. Sets the pen color to blue (RGB: 0,0,255)
  3. Moves to position (100,100) and draws a circle with radius 50
  4. Changes the pen color to red (RGB: 255,0,0)
  5. Moves to position (150,50) and draws a 100x100 rectangle

Architecture

This BOOSE interpreter is built in C# using Windows Forms and follows object-oriented design principles:

  • Command Pattern: Each BOOSE command is implemented as a separate command object
  • Factory Pattern: Commands are created using a factory for flexibility and extensibility
  • Singleton Pattern: Debug window uses singleton pattern for centralized logging
  • Parser: Converts BOOSE source code into executable command objects
  • Canvas: Abstracts graphics operations for drawing output

Use Cases

BOOSE is ideal for:

  • Learning programming fundamentals in a visual way
  • Teaching object-oriented design patterns
  • Understanding interpreter and compiler construction
  • Practicing software engineering principles (testing, documentation, version control)
  • Experimenting with language design and implementation

Project Structure

The BOOSE interpreter consists of:

  • BOOSE.Interpreter: Main interpreter implementation with parser, canvas, and UI
  • BOOSE.Interpreter.Tests: Unit tests for commands and interpreter functionality
  • BOOSE.dll: Base library providing core parsing and command infrastructure
  • BOOSEexamplePrograms: Sample programs demonstrating BOOSE features
